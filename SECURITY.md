# Security Policy

## Supported Versions

As deer-flow doesn't provide an official release yet, please use the latest version for security updates.
Currently, we maintain two active branches:
* `main` branch for deer-flow 2.x
* `main-1.x` branch for deer-flow 1.x

| Version | Branch       | Supported          |
|---------|--------------|--------------------|
| 2.x     | `main`       | :white_check_mark: |
| 1.x     | `main-1.x`   | :white_check_mark: |

## Reporting a Vulnerability

Please report security issues via [GitHub Security Advisories](https://github.com/bytedance/deer-flow/security/advisories/new).

For the best response, please include:
- Description of the vulnerability and its impact
- Steps to reproduce
- Affected component(s) and version(s)
- Any potential mitigations or patches you've identified

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Fix Timeline**: Depends on severity; critical issues are addressed as a priority

## Security Scope

This policy covers the deer-flow core application. Third-party integrations (LLM providers, tool services, etc.) may have their own security policies. Please report issues with external services to their respective maintainers.
