# DeerFlow 后端代理系统文档

## 📋 文档说明

本文档作为后端代理系统（Agent System）的入口文档，指向后端架构和设计模式的核心文档。

### 什么是代理系统？
在 DeerFlow 中，**代理**是整个框架的核心执行单元。它不是简单的聊天机器人，而是一个能够：
- **理解用户意图**：通过 LLM 解析用户的复杂请求
- **规划执行路径**：自动决定需要调用哪些工具和技能
- **协调多系统协作**：调度工具、技能、沙箱、记忆等子系统
- **处理错误和重试**：当某个步骤失败时，智能调整策略
- **流式返回结果**：实时向用户反馈执行进度

## 📝 补充说明

**为什么这样设计**：
- **保持文档结构的模块化**：避免在多个文件中重复相同的内容，降低维护成本
- **将架构设计集中在一个权威文档中**：CLAUDE.md 是架构的"单一真实来源"（Single Source of Truth）
- **便于维护和更新**：当架构变更时，只需更新 CLAUDE.md，所有引用它的文档自动获得最新信息

**设计思路**：
- **使用相对路径引用**：确保文档的可移植性，即使项目被克隆到不同位置也能正常工作
- **简洁的入口文档**：快速引导开发者找到核心内容，而不是在这里展开所有细节
- **符合文档最佳实践**：入口文档应该是"地图"而非"百科全书"，让读者快速导航到目的地

**文件组织原则**：
```
AGENTS.md (入口) → CLAUDE.md (详细架构) → 具体模块文档
     ↑                      ↑                    ↑
  快速导航          核心设计决策           实现细节
```
