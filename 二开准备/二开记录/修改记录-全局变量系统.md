# 全局变量系统开发记录

**日期**: 2026-04-27
**功能**: 全局变量系统（动态提示词注入）
**开发者**: AI Assistant
**版本**: v1.0（一期）+ v2.0（二期系统变量+模板替换）

---

## 一、需求背景

DeerFlow 项目需要在每次调用大模型时，**动态向系统提示词中注入可变的变量值**。

### 一期需求
1. **增删改查**：大模型可通过工具修改变量；外部业务脚本可通过 API 修改变量
2. **两级隔离**：项目级变量（所有对话共享）+ 对话级变量（单个对话独立）
3. **自动注入**：变量值自动全部注入到系统提示词中

### 二期需求
1. **模板替换**：不再自动注入，改为在 SOUL.md 等提示词模板中用 `{{变量名}}` 语法手动引用
2. **系统变量**：新增默认的系统变量（如 `workdir`），前端不可编辑、大模型不可修改
3. **llm_editable 权限**：每个变量可控制大模型是否能通过工具修改
4. **前端管理页面**：在系统设置中新增"全局变量"管理页面

### 2.2 系统变量

```python
# 定义在 storage.py 中
SYSTEM_VARIABLES = {
    "workdir": {
        "value": "/mnt/shared-data",
        "description": "Shared workspace directory",
        "is_system": True,
        "llm_editable": False,
        "updated_at": "system",
        "updated_by": "system",
    },
}
```

**特性**：
- 默认存在，每个用户都有
- 前端展示但不可编辑（显示锁定图标）
- 大模型工具无法修改或删除
- 保存时自动过滤，不写入文件
- 加载时自动合并到变量列表中

### 2.3 模板替换

**语法**：`{{变量名}}`

**示例**（SOUL.md）：
```markdown
当前工作目录：{{workdir}}
你是{{role}}，当前处理第{{chapter}}章。
```

**替换规则**：
1. 对话级变量覆盖项目级变量
2. 未匹配的 `{{变量名}}` 保持原样不替换
3. 替换发生在两处：
   - **加载时**：`get_agent_soul()` 读取 SOUL.md 后替换（项目级变量）
   - **运行时**：中间件在每次大模型调用前替换（含对话级变量）

### 2.4 权限模型

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `is_system` | bool | false | 系统变量（前端只读、LLM 不可修改） |
| `llm_editable` | bool | true | 大模型是否可通过工具修改 |

**权限矩阵**：

| 操作 | 系统变量 | 自定义+llm_editable=true | 自定义+llm_editable=false |
|------|---------|--------------------------|---------------------------|
| 前端查看 | ✅ | ✅ | ✅ |
| 前端编辑 | ❌ | ✅ | ✅ |
| 前端删除 | ❌ | ✅ | ✅ |
| LLM 读取 | ✅ | ✅ | ✅ |
| LLM 修改 | ❌ | ✅ | ❌ |
| LLM 删除 | ❌ | ✅ | ❌ |

## 四、使用方法

### 4.1 前端配置

1. 打开 **设置对话框**（右上角齿轮图标）
2. 选择 **全局变量** 菜单项
3. 默认显示项目级变量，可通过下拉切换到对话级
4. 点击 **+ 添加变量** 按钮创建新变量
5. 系统变量（如 `workdir`）以灰色锁定图标显示，不可编辑

### 4.2 在提示词中引用变量

在 SOUL.md 或 SKILL.md 中使用 `{{变量名}}` 语法：

```markdown
## 角色
你是{{role}}，专注于{{domain}}领域。

## 工作目录
当前工作目录：{{workdir}}

## 当前任务
正在处理第{{chapter}}章的内容。
```

### 4.3 大模型修改变量

大模型可通过工具修改当前对话的变量：

```
set_variable(key="chapter", value="6", scope="thread", description="当前章节")
```

### 4.4 外部脚本调用 API

```bash
# 设置项目级变量
curl -X PUT http://localhost:8000/api/global-variables/project/mode \
  -H "Content-Type: application/json" \
  -d '{"value": "writing", "description": "工作模式", "llm_editable": true}'

# 查看变量
curl http://localhost:8000/api/global-variables/project

# 删除变量
curl -X DELETE http://localhost:8000/api/global-variables/project/mode
```

## 七、注意事项

1. **变量引用语法**：使用 `{{变量名}}`（双大括号），不是 `${变量名}`（美元符号）
2. **变量名**：系统变量名是 `workdir`（小写），不是 `__WORKING_DIR__`
3. **模板替换时机**：SOUL.md 加载时替换（项目级变量）+ 运行时中间件替换（含对话级变量）
4. **系统变量保护**：前端只显示锁定图标，API 返回 403 拒绝修改
