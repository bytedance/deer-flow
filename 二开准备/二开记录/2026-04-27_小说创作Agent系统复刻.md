## 三、问题修复记录

### 问题1: API返回403错误
- **现象**: `net::ERR_ABORTED http://localhost:2026/api/agents`
- **原因**: `agents_api.enabled: false`
- **修复**: 改为 `true` 并重启服务

### 问题2: Skills返回空数组
- **现象**: Skills API返回空数组
- **原因**: `skills.path: ../skills` 路径解析错误
- **修复**: 改为 `skills.path: skills`

### 问题3: ModuleNotFoundError: No module named 'my_tools'
- **现象**: LangGraph日志报模块找不到错误，历史对话加载失败
- **原因**: LangGraph进程无法访问repo根目录
- **修复**: 在serve.sh中添加 `PYTHONPATH="$REPO_ROOT":.`

### 问题4: SKILL.md名称格式错误
- **现象**: Skills无法被正确发现
- **原因**: 名称使用下划线（`novel_anti_ai_detector`）
- **修复**: 改为连字符（`novel-anti-ai-detector`）

### 问题5: post_write_validator.py语法错误
- **现象**: Python导入失败
- **原因**: 中文引号导致字符串终止
- **修复**: 重写该文件，修正引号问题

### 问题6: novel-master无法调用子Agent（task工具不可用）
- **现象**: 无论调用哪个子Agent，系统返回 `Error: agent is not a valid tool`，可用工具列表中不包含task工具
- **原因分析**:
  1. 前端只在 `mode === "ultra"` 时才传递 `subagent_enabled: true`，其他模式下task工具不会被加载
  2. novel-master的SOUL.md中使用了错误的工具调用语法（`Agent`工具而非`task`工具）
  3. AgentConfig类不支持`subagent_enabled`字段，无法在agent配置层面强制启用task工具
- **修复方案**:
  1. **扩展AgentConfig类**（`backend/packages/harness/deerflow/config/agents_config.py`）:
     - 添加 `subagent_enabled: bool = False` 字段
     - 添加 `max_concurrent_subagents: int = 3` 字段
  2. **修改make_lead_agent函数**（`backend/packages/harness/deerflow/agents/lead_agent/agent.py`）:
     - 当agent_config的subagent_enabled为True时，强制启用task工具，覆盖前端传递的值
  3. **修复novel-master的SOUL.md**:
     - 将错误的`Agent`工具调用语法改为正确的`task`工具调用语法
     - 添加task工具的参数说明（description, prompt, subagent_type, max_turns）
     - 列出所有14个可用子Agent及其功能
  4. **novel-master的config.yaml添加配置**:
     ```yaml
     subagent_enabled: true
     max_concurrent_subagents: 3
     ```
  5. **扩展Gateway API响应模型**（`backend/app/gateway/routers/agents.py`）:
     - `AgentResponse` 添加 `subagent_enabled` 和 `max_concurrent_subagents` 字段
     - `_agent_config_to_response` 返回这两个字段
- **受影响的文件**:
  - `backend/packages/harness/deerflow/config/agents_config.py` - 扩展AgentConfig
  - `backend/packages/harness/deerflow/agents/lead_agent/agent.py` - 修改make_lead_agent
  - `backend/app/gateway/routers/agents.py` - 扩展API响应模型
  - `backend/.deer-flow/agents/novel-master/config.yaml` - 添加subagent_enabled
  - `backend/.deer-flow/agents/novel-master/SOUL.md` - 修正工具调用语法

## 六、如何配置主Agent调用子Agent

本章节详细说明如何让一个主Agent通过task工具调用子Agent。

### 1. 框架层修改（一次性）

#### 1.1 扩展AgentConfig类

**文件**: `backend/packages/harness/deerflow/config/agents_config.py`

```python
class AgentConfig(BaseModel):
    """Configuration for a custom agent."""

    name: str
    description: str = ""
    model: str | None = None
    tool_groups: list[str] | None = None
    skills: list[str] | None = None
    # 新增：控制是否启用task工具调用子Agent
    subagent_enabled: bool = False
    # 新增：最大并发子Agent任务数
    max_concurrent_subagents: int = 3
```

#### 1.2 修改make_lead_agent函数

**文件**: `backend/packages/harness/deerflow/agents/lead_agent/agent.py`

在 `make_lead_agent` 函数中添加以下逻辑，使agent配置可以覆盖前端传递的subagent_enabled：

```python
# Agent config can force subagent_enabled and max_concurrent_subagents
# This is useful for agents like novel-master that need task tool regardless of frontend mode
if agent_config:
    if agent_config.subagent_enabled:
        subagent_enabled = True
        max_concurrent_subagents = agent_config.max_concurrent_subagents
```

#### 1.3 扩展Gateway API响应模型

**文件**: `backend/app/gateway/routers/agents.py`

扩展 `AgentResponse` 模型，添加subagent_enabled和max_concurrent_subagents字段：

```python
class AgentResponse(BaseModel):
    """Response model for a custom agent."""

    name: str
    description: str
    model: str | None
    tool_groups: list[str] | None
    skills: list[str] | None
    soul: str | None
    subagent_enabled: bool = Field(default=False)           # 新增
    max_concurrent_subagents: int = Field(default=3)        # 新增
```

在 `_agent_config_to_response` 函数中返回这两个字段：

```python
return AgentResponse(
    name=agent_cfg.name,
    description=agent_cfg.description,
    model=agent_cfg.model,
    tool_groups=agent_cfg.tool_groups,
    skills=agent_cfg.skills,
    soul=soul,
    subagent_enabled=agent_cfg.subagent_enabled,              # 新增
    max_concurrent_subagents=agent_cfg.max_concurrent_subagents,  # 新增
)
```

### 3. 主Agent的SOUL.md指令

**文件**: `backend/.deer-flow/agents/{agent-name}/SOUL.md`

在SOUL.md中说明如何使用task工具调用子Agent：

```markdown
## 子Agent调用规范

### 调用格式

使用 `task` 工具调用子Agent，参数如下：
- `description`：简短任务描述（3-5词），用于日志显示
- `prompt`：给子Agent的详细任务指令
- `subagent_type`：子Agent名称（如novel-writer、outline-planner等）
- `max_turns`：（可选）最大回合数

### 示例

调用 novel-writer 写第5章：
- description: "Write chapter 5"
- prompt: "请根据 book/小说名/02-正文/第5章/_task/写作任务汇总.md 中的内容，撰写第5章正文。输出到 book/小说名/02-正文/第5-8章/第5章.md"
- subagent_type: "novel-writer"

### 可用子Agent

**规划类**：
- `novel-architect`：生成完整基础设定
- `volume-planner`：生成卷纲和分卷计划
- `outline-planner`：生成或修改细纲
- `book-rules-manager`：管理书籍规则

**整理类**：
- `novel-world-organizer`：整理世界观
- `novel-character-organizer`：整理人物
- `novel-item-organizer`：整理道具

**写作类**：
- `novel-writer`：根据汇总内容写章节正文
- `continuity-auditor`：审核章节连续性
- `novel-reviser`：修稿编辑

**状态管理类**：
- `state-settler`：结算器
- `chapter-summarizer`：章节总结
- `card-manager`：名片管理
- `hook-manager`：伏笔管理

### 上下文传递规则

1. **不传历史对话**：所有子Agent调用时不fork历史对话
2. **通过文件传递**：上下文通过文件路径传递，子Agent自行读取
3. **关键信息提取**：对于简单任务，主控可以直接提取关键信息传给子Agent
```

### 4. 子Agent配置格式

**文件**: `config.yaml` → `subagents.custom_agents`

每个子Agent需要以下配置：

```yaml
subagents:
  custom_agents:
    novel-writer:
      # description: 何时委托给此子Agent（必需）
      description: |
        长篇网文写手，负责根据写作任务汇总撰写小说正文。
        读取细纲、世界观、人物、道具等参考信息写正文。
        当需要写小说章节时调用此Agent。
      
      # system_prompt: 指导子Agent行为的系统提示（必需）
      system_prompt: |
        你是一个专业的长篇网文写手，负责根据写作任务汇总文件中的细纲、世界观、人物、道具等参考信息，撰写高质量的小说正文。

        ## 写作风格要求
        1. 用口语化的对白推进剧情
        2. 强台词、强情绪、强冲突、强对抗、强矛盾
        ...

      # tools: 工具白名单（可选，None=继承父Agent所有工具）
      tools:
        - read_file
        - write_file
      
      # model: 使用的模型，inherit表示继承父Agent的模型（默认"inherit"）
      model: inherit
      
      # max_turns: 最大执行回合数（默认50）
      max_turns: 80
      
      # timeout_seconds: 超时时间，单位秒（默认900）
      timeout_seconds: 1800
```

### 6. 关键注意事项

1. **subagent_enabled必须在主Agent的config.yaml中设置为true**，否则task工具不会被加载
2. **前端mode不影响主Agent的task工具**：即使前端使用standard/flash模式，主Agent依然可以调用子Agent
3. **子Agent不会继承task工具**：子Agent的task工具会被自动禁用（subagent_enabled=False），防止无限嵌套
4. **SOUL.md中的工具名称必须正确**：使用 `task` 而非 `Agent` 或 `agent`
5. **每个子Agent必须有description和system_prompt**：缺少任一字段会导致子Agent无法注册
