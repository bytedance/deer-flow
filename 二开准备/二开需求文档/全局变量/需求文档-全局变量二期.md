## 一、业务需求

### 1.1 前端：全局变量管理页面

在系统设置中新增"全局变量"管理页面，让用户可以通过可视化界面管理全局变量。

**核心需求**：
- 页面放在"设置对话框"的侧边栏导航中
- 默认显示**项目级变量**，可通过下拉切换**对话级变量**
- 支持变量的**增、删、改、查**
- 变量包含：变量名（key）、变量值（value）、描述（description）、**大模型可编辑（llm_editable）**
- 可以修改变量名称（重命名）
- 可以手动修改变量值
- 可以控制大模型是否能通过工具修改该变量

### 1.2 后端：模板替换

将一期中"自动全部注入"的方式改为**模板替换**方式。

**核心需求**：
- 不再自动把变量块塞入系统提示词
- 在加载 SOUL.md、SKILL.md 等提示词内容时，自动将其中的 `{{变量名}}` 替换为实际变量值
- 例：SOUL.md 中写 `你是{{role}}，当前处理第{{chapter}}章`
- 系统自动替换为 `你是写作助手，当前处理第5章`

### 1.3 后端：大模型修改变话级变量工具

新增一个 LangChain tool，让大模型在对话中可以修改**当前对话**的全局变量。

**核心需求**：
- 大模型可以通过工具修改当前对话的变量
- 修改变量后立即生效，下次提示词组装时使用新值

#### 3.1.4 交互说明

**添加变量**：
1. 点击"+ 添加变量"按钮
2. 弹出对话框，输入：变量名、变量值、描述（选填）、**大模型可编辑（默认开启）**
3. 变量名校验：非空、最长100字符、不能包含特殊字符
4. 变量值校验：非空、最长500字符
5. 保存成功后刷新列表

**修改变量**：
1. 点击行内编辑按钮（✏️）
2. 弹出对话框，预填当前值
3. 可修改变量名（重命名）、变量值、描述、**大模型可编辑开关**
4. 修改变量名时，相当于删除旧变量+创建新变量

**删除变量**：
1. 点击行内删除按钮（🗑️）
2. 弹出确认对话框
3. 确认后删除

**切换作用域**：
1. 下拉框选择"项目级"或"对话级"
2. 切换后刷新列表
3. 对话级需要额外选择具体对话（thread_id）

### 3.2 后端：模板替换

#### 3.2.1 实现位置

`backend/packages/harness/deerflow/global_variables/prompt_injector.py`

新增函数：

```python
def replace_template_variables(text: str, thread_id: str | None = None) -> str:
    """将文本中的 {{变量名}} 替换为实际变量值。

    变量覆盖规则：对话级 > 项目级
    未匹配的 {{变量名}} 保持原样不替换。
    """
```

#### 3.2.2 调用点

| 函数 | 文件 | 修改内容 |
|------|------|----------|
| `get_agent_soul()` | `prompt.py` | 读取 SOUL.md 后调用 `replace_template_variables()` |
| `get_skills_prompt_section()` | `prompt.py` | 读取 SKILL.md 后调用 `replace_template_variables()` |

#### 3.2.3 替换逻辑

```
1. 读取原始文本（SOUL.md / SKILL.md 等）
2. 获取合并后的变量（项目级 + 对话级覆盖）
3. 用正则匹配 {{变量名}} 模式
4. 如果变量存在，替换为变量值
5. 如果变量不存在，保持原样不替换
6. 返回替换后的文本
```

#### 3.2.4 示例

SOUL.md 原始内容：
```markdown
# 角色
你是{{role}}，专注于{{domain}}领域。

## 当前任务
正在处理第{{chapter}}章的内容。
```

假设变量值为：
- role = "写作助手"
- chapter = "5"
- domain 变量不存在

替换后：
```markdown
# 角色
你是写作助手，专注于{{domain}}领域。

## 当前任务
正在处理第5章的内容。
```

### 3.3 后端：大模型修改变话级变量工具

一期已实现 `set_variable_tool`，工具会自动从 runtime 中获取 thread_id。

**二期新增**：工具操作前检查 `llm_editable` 权限。

```python
@tool("set_variable", parse_docstring=True)
def set_variable_tool(...):
    """Set the value of a global variable.
    
    Args:
        key: The variable key to set.
        value: The variable value.
        scope: The scope of the variable. Must be 'project' or 'thread'.
        description: Optional description of the variable.
    """
    # 加载变量数据
    data = get_storage().load(scope, thread_id=thread_id)
    variables = data.get("variables", {})
    existing_var = variables.get(key, {})
    
    # 检查 llm_editable 权限
    if isinstance(existing_var, dict):
        llm_editable = existing_var.get("llm_editable", True)
        if not llm_editable:
            return f"Error: variable '{key}' cannot be modified by the AI assistant"
    
    # 正常设置流程...
```

## 七、全局变量存储位置

当前全局变量存储在**文件系统**中，不使用数据库：

| 级别 | 存储路径 | 格式 |
|------|----------|------|
| 项目级 | `{DEER_FLOW_HOME}/global_variables.json` | JSON |
| 对话级 | `{DEER_FLOW_HOME}/threads/{thread_id}/variables.json` | JSON |

`DEER_FLOW_HOME` 默认为用户主目录下的 `.deer-flow` 文件夹。可通过环境变量 `DEER_FLOW_HOME` 修改。

数据结构示例（二期新增 `llm_editable` 字段）：

```json
{
  "variables": {
    "mode": {
      "value": "writing",
      "description": "工作模式",
      "llm_editable": true,
      "updated_at": "2025-04-27T10:00:00Z",
      "updated_by": "api"
    },
    "api_key": {
      "value": "sk-xxx",
      "description": "API密钥",
      "llm_editable": false,
      "updated_at": "2025-04-27T10:00:00Z",
      "updated_by": "api"
    }
  },
  "lastUpdated": "2025-04-27T10:00:00Z"
}
```

**`llm_editable` 字段说明**：
- `true`（默认）：大模型可以通过工具修改变量
- `false`：大模型无法通过工具修改，只能前端/API 修改
- 一期创建的变量默认 `llm_editable = true`，向后兼容

前端通过一期已实现的 REST API 读写这些文件，无需直接访问文件系统。
