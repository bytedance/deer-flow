# 前端需求文档：对话页面文件浏览器

> 目标：在现有 Artifact 系统基础上扩展，支持浏览 AI 工作目录、编辑和删除文件

---

## 1. 需求概述

### 1.1 功能目标
扩展现有的 Artifact 系统，增加：
- 浏览 AI 工作目录（`/mnt/user-data/workspace`）下的文件夹和文件
- 文件夹可展开/折叠，只显示 `.md/.txt/.json/.log` 格式的文件
- 点击文件可查看、编辑内容
- 支持保存和删除文件
- 复用现有的文件预览功能（ArtifactFileDetail）

### 1.2 界面位置
```
┌──────────────────────────────────────────────────────────┐
│              Artifact 面板（右侧边栏）                    │
│  ┌────────────────────────────────────────────────────┐  │
│  │  📁 folder1/                                       │  │
│  │  📁 folder2/                                       │  │
│  │  📄 notes.md     [编辑] [删除]                      │  │
│  │  📄 log.txt      [编辑] [删除]                      │  │
│  └────────────────────────────────────────────────────┘  │
│  ┌────────────────────────────────────────────────────┐  │
│  │           文件查看/编辑区域                         │  │
│  │           (复用 ArtifactFileDetail)                 │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

## 4. 组件修改清单

### 4.1 ArtifactFileList 组件修改

**新增功能：**
- 显示目录树结构（不只是文件列表）
- 文件夹可展开/折叠
- 加载工作目录内容（通过 API）
- 只显示 `.md/.txt/.json/.log` 文件
- 每个文件显示编辑和删除按钮

**接口变化：**
```typescript
interface ArtifactFileListProps {
  className?: string
  files?: string[]              // 现有：AI 生成的文件列表
  threadId: string
  showDirectoryBrowser?: boolean // 新增：是否显示目录浏览
}
```

### 4.2 ArtifactFileDetail 组件修改

**新增功能：**
- 支持编辑模式（不只是只读查看）
- 添加保存按钮
- 添加删除按钮
- 显示文件路径
- 检测文件是否有未保存的更改

**新增 Props：**
```typescript
interface ArtifactFileDetailProps {
  // 现有 props 保持不变
  className?: string
  filepath: string
  threadId: string
  
  // 新增
  editable?: boolean            // 是否允许编辑
  onSave?: (content: string) => Promise<void>  // 保存回调
  onDelete?: () => Promise<void>              // 删除回调
}
```

### 4.3 ArtifactsProvider 修改

**新增状态：**
```typescript
interface ArtifactsContextType {
  // 现有状态保持不变
  artifacts: string[]
  setArtifacts: (artifacts: string[]) => void
  selectedArtifact: string | null
  select: (artifact: string, autoSelect?: boolean) => void
  open: boolean
  setOpen: (open: boolean) => void
  
  // 新增：工作目录浏览状态
  directoryEntries: DirectoryEntry[]     // 目录条目
  expandedFolders: Set<string>           // 展开的文件夹
  currentPath: string                    // 当前浏览路径
  setDirectoryEntries: (entries: DirectoryEntry[]) => void
  toggleFolder: (path: string) => void
  loadDirectory: (path: string) => Promise<void>
}

interface DirectoryEntry {
  name: string
  path: string
  isDirectory: boolean
}
```

## 5. 交互流程

### 5.1 浏览目录
```
1. 用户点击 Artifact 按钮打开面板
2. 显示工作目录内容（/mnt/user-data/workspace）
3. 文件夹可展开/折叠
4. 只显示 .md/.txt/.json/.log 文件
5. 每个文件有编辑和删除按钮
```

### 5.2 查看/编辑文件
```
1. 点击文件名 → 打开 ArtifactFileDetail
2. 默认显示只读模式（使用现有功能）
3. 点击编辑按钮 → 切换为编辑模式
4. CodeEditor 变为可编辑
5. 修改内容后显示"未保存"提示
6. 点击保存 → 调用 API 保存
7. 保存成功后提示成功
```

### 5.3 删除文件
```
1. 点击文件旁边的删除按钮
2. 弹出 Dialog 确认
3. 确认后调用 API 删除
4. 删除成功后：
   - 从目录列表中移除
   - 如果当前正在查看该文件，关闭查看面板
   - 显示成功提示
```

## 6. API 接口

### 6.1 调用方式

前端直接调用 Python 后端的 FastAPI 接口：

```typescript
// 后端 API 地址通过环境变量配置
const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL || 'http://localhost:8000'

// 浏览目录
GET ${API_BASE_URL}/api/filesystem/browse?thread_id={threadId}&path={virtualPath}&extensions=md,txt,json,log

// 读取文件
GET ${API_BASE_URL}/api/filesystem/read?thread_id={threadId}&path={virtualPath}

// 写入文件
POST ${API_BASE_URL}/api/filesystem/write
Body: { thread_id: string, path: string, content: string }

// 删除文件
DELETE ${API_BASE_URL}/api/filesystem/delete?thread_id={threadId}&path={virtualPath}
```

### 6.2 前端 API 客户端

```typescript
// core/filesystem/api.ts

export async function browseDirectory(
  threadId: string,
  path: string,
  extensions?: string[]
): Promise<DirectoryEntry[]>

export async function readFile(
  threadId: string,
  path: string
): Promise<{ path: string, content: string }>

export async function writeFile(
  threadId: string,
  path: string,
  content: string
): Promise<void>

export async function deleteFile(
  threadId: string,
  path: string
): Promise<void>
```

## 7. 样式要求

### 7.1 目录树样式
- 使用现有的 Card 组件
- 文件夹图标：lucide-react 的 `Folder` / `FolderOpen`
- 文件图标：复用 `getFileIcon`
- 缩进表示层级
- 悬停显示编辑/删除按钮

### 7.2 编辑模式样式
- 复用现有的 CodeEditor 组件
- 顶部工具栏显示：
  - 返回按钮
  - 文件名
  - 保存按钮
  - 删除按钮
- 未保存时显示指示器（圆点或"未保存"文字）

## 8. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 目录加载失败 | 显示错误提示，提供重试按钮 |
| 文件读取失败 | 显示错误信息，返回列表 |
| 保存失败 | 显示 Toast 错误，保留编辑状态 |
| 删除失败 | 显示 Toast 错误 |
| 工作目录不存在 | 显示空状态提示 |
