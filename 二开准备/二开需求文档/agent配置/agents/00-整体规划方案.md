# 本期 Agent 规划 - 小说创作系统融合方案

> 融合两套现有 Agent 体系，打造完整的小说创作自动化系统
>
> - 来源1：老 Agent（数据库查询助理类，11个提示词）
> - 来源2：其他项目 Agent（完整小说创作系统，16个 Agent）

***

## 1. 整体架构

```
主 Agent (Master)
│
├── 规划流程 Agent
│   ├── Architect（架构师）- 新建小说基础设定
│   ├── VolumePlanner（卷纲规划师）- 卷纲编写/修改
│   ├── OutlinePlanner（细纲规划师）- 章节细纲编写/修改
│   └── BookRulesManager（规则管理器）- 本书规则管理
│
├── 整理流程 Agent（写作前准备）
│   ├── WorldOrganizer（世界观整理）- 整理当前世界观设定
│   ├── CharacterOrganizer（人物整理）- 整理人物档案和关系
│   ├── ItemOrganizer（道具整理）- 整理道具和技能信息
│   ├── StorylineOrganizer（故事线整理）- 整理故事走向和章节路径
│   └── ContextAssembler（上下文组装）- Tool，将各整理结果组装为写作任务 MD
│
├── 写作流程 Agent
│   ├── Writer（写手）- 读取任务 MD，写正文
│   ├── ContinuityAuditor（连续性审计师）- 多维度审核
│   ├── AntiAIDetector（AI 痕迹检测）- Skill，去除 AI 味
│   ├── PlanComplianceChecker（规划遵循检查）- Skill，检查是否遵循规划
│   ├── Reviser（修稿编辑）- 根据审核结果修改
│   └── OutlineSyncer（细纲同步器）- 根据实际写作同步修改细纲
│
├── 状态管理 Agent
│   ├── Settler（结算器）- 更新状态文件
│   ├── ChapterSummarizer（章节总结）- 生成章节摘要
│   ├── CardManager（名片管理）- 更新小说名片（JSON）
│   └── HookManager（伏笔管理）- 管理伏笔池
│
└── 辅助工具 Agent
    ├── ChapterAnalyzer（章节分析师）- 导入已有章节
    ├── Consolidator（整合师）- 压缩摘要
    ├── Radar（雷达）- 市场分析
    └── LengthNormalizer（长度规范化器）- 字数调整
```

## 2. 目录结构

每本小说在 `book/` 下创建独立文件夹：

```
book/[小说名称]/
├── card.json                    # 小说名片（JSON 格式，记录进度等元数据）
├── 00-世界观/
│   ├── story-bible.md           # 故事圣经（世界观、力量体系、核心冲突）
│   ├── character-matrix.md      # 角色矩阵（角色档案、关系网）
│   ├── subplot-board.md         # 支线板（多条故事线跟踪）
│   └── emotional-arcs.md        # 情感弧线（角色情感发展）
├── 01-规划/
│   ├── volume-outline.md        # 卷纲（分卷概览 + 章节分组规划）
│   ├── book-rules.json          # 本书规则（JSON 格式，硬规则+风格指南）
│   ├── book-plan.md             # 创作计划
│   └── chapters/                # 章节细纲（每 5 章一组）
│       ├── 第01-05章-细纲.md
│       ├── 第06-10章-细纲.md
│       └── ...
├── 02-正文/                     # 正文按章节组组织
│   └── 第N-M章/                 # 每组一个文件夹（如：第01-05章/）
│       ├── _task/               # 临时任务目录（写作时创建，完成后清理）
│       │   ├── 世界观参考.md
│       │   ├── 人物参考.md
│       │   ├── 道具参考.md
│       │   ├── 故事线参考.md
│       │   ├── 用户要求.md
│       │   └── 写作任务汇总.md
│       ├── 第N章.md             # 各章节正文
│       ├── 第N+1章.md
│       └── ...
├── 03-状态/
│   ├── current-state.md         # 当前状态卡（主角位置、目标、敌人等）
│   ├── pending-hooks.md         # 伏笔池（未解决伏笔跟踪）
│   └── chapter-summaries.md     # 章节摘要汇总
├── 04-审稿/
│   ├── 第01章-审计报告.md
│   ├── 第01章-修改记录.md
│   └── ...
├── 05-参考/
│   ├── style-profile.md         # 风格指纹（从样章提取）
│   └── market-radar.md          # 市场分析（如适用）
└── 06-归档/
    ├── consolidated-summaries.md # 压缩后的卷摘要
    └── 历史版本/                 # 重要修改前备份
```

### 2.1 写作任务临时目录

当用户要求写某一章时，主 Agent 会在对应章节组文件夹下创建 `_task/` 文件夹：

```
book/[小说名称]/
└── 02-正文/
    └── 第N-M章/                 # 章节组文件夹（如：第01-05章/）
        ├── _task/               # 临时任务目录
        │   ├── 世界观参考.md    # WorldOrganizer 输出
        │   ├── 人物参考.md      # CharacterOrganizer 输出
        │   ├── 道具参考.md      # ItemOrganizer 输出
        │   ├── 故事线参考.md    # StorylineOrganizer 输出
        │   ├── 用户要求.md      # 用户的写作指令
        │   └── 写作任务汇总.md   # ContextAssembler 组装的最终任务文件
        ├── 第N章.md             # 章节正文
        └── ...
```

<br />

### 2.2 文件类型说明

| 文件类型        | 格式      | 说明                                 |
| ----------- | ------- | ---------------------------------- |
| 大部分设定/规划/正文 | `.md`   | Markdown 格式                        |
| 小说名片        | `.json` | JSON 格式，包含进度、统计等结构化数据，方便代码读取       |
| 本书规则        | `.json` | JSON 格式，硬规则、风格指南、约束，方便代码和 Agent 读取 |

## 3. 主 Agent (Master)

### 3.1 基本信息

| 字段 | 值                                                                        |
| -- | ------------------------------------------------------------------------ |
| 名称 | `novel-master`                                                           |
| 描述 | 小说创作系统主控 Agent，负责任务分发、流程协调、结果汇总                                          |
| 模型 | sonnet 或 opus                                                            |
| 工具 | Bash, FileRead, FileEdit, FileWrite, Glob, Grep, Agent, Skill, TodoWrite |
| 记忆 | project                                                                  |

### 3.2 核心职责

1. **任务接收与分析**：理解用户需求，判断任务类型（新建/写作/修改/规划）
2. **流程编排**：按任务类型调用正确的 Agent 执行顺序
3. **上下文管理**：为子 Agent 传递必需的上下文文件，**不传历史对话**
4. **文件管理**：负责读取和保存所有 Agent 的输出到正确位置
5. **质量控制**：验证子 Agent 输出，不合格时要求重做
6. **进度汇报**：向用户汇报当前进度和结果

### 3.3 核心规则

- **不传历史对话**：调用子 Agent 时只传递当前任务必需的上下文文件
- **文件路径固定**：始终从 `book/[小说名称]/` 下读写
- **临时目录清理**：写作任务完成后可选择清理 `_task/` 目录
- **名片更新**：每次章节完成后更新 `card.json`

### 3.4 工作流程

#### 流程 1：新建小说

```
用户："我要写一本新小说，书名XXX，类型XXX"
   ↓
1. 创建完整目录结构
   ↓
2. 创建 card.json（初始状态）
   ↓
3. 调用 Architect 生成基础设定
   ├─ story-bible.md
   ├─ volume-outline.md
   ├─ book-rules.json
   ├─ book-plan.md
   ├─ current-state.md
   └─ pending-hooks.md
   ↓
4. 汇报完成，引导下一步
```

#### 流程 2：写正文（单章，含整组细纲传递）

```
用户："帮我写第N章，剧情是..."
   ↓
1. 创建 `02-正文/第N-M章/_task/` 临时目录
   ↓
2. 并行调用整理 Agent：
   ├─ WorldOrganizer → `_task/世界观参考.md`
   ├─ CharacterOrganizer → `_task/人物参考.md`
   ├─ ItemOrganizer → `_task/道具参考.md`
   └─ StorylineOrganizer → `_task/故事线参考.md`
   ↓
3. 将用户要求写入 `_task/用户要求.md`
   ↓
4. 调用 ContextAssembler：
   读取所有整理结果 + 用户要求 + 整组细纲（5章）
   → 组装为 `_task/写作任务汇总.md`
   ↓
5. 调用 Writer：
   读取 `写作任务汇总.md`（含整组细纲）
   → 从中提取第N章细纲
   → 写正文 → `02-正文/第N-M章/第N章.md`
   ↓
6. 调用审核流程（见流程 4）
   ↓
7. 调用 Settler 更新状态
   ↓
8. 更新 card.json（进度+1）
   ↓
9. 可选择清理 `_task/` 目录
   ↓
10. 汇报完成

（写第N+1章时，重复上述流程，传递同样的整组细纲）
```

#### 流程 3：写正文（继续往后写）

```
用户："继续写下一章" 或 "帮我写第N章"
   ↓
1. 读取 card.json 获取当前进度
   ↓
2. 调用各种整理 Agent（同流程 2）
   └─ 整理世界观、人物、道具、故事走向、章节路径等
   ↓
3. ContextAssembler 组装写作任务
   ↓
4. Writer 写正文
   ↓
5. 审核流程
   ↓
6. 状态更新
   ↓
7. 更新 card.json
```

#### 流程 4：审核与修改

```
（在 Writer 完成后自动触发）
   ↓
1. 运行 PostWriteValidator（Skill，正则检查格式）
   ↓
2. 调用 ContinuityAuditor：
   多维度审计 → book/[小说名称]/04-审稿/第N章-审计报告.md
   ↓
3. 运行 AntiAIDetector（Skill，AI 痕迹检测）
   ↓
4. 运行 PlanComplianceChecker（Skill，规划遵循检查）
   ↓
5. 判断审核结果：
   ├─ 通过 → 进入流程 6
   └─ 未通过 →
       a. 审计报告已保存
       b. 调用 Reviser 修改（传入审计报告）
       c. 修改后重新审核（最多 3 轮）
       d. 如仍不通过，报告用户并询问
   ↓
6. 调用 OutlineSyncer：
   根据实际写作内容同步修改细纲
   → book/[小说名称]/01-规划/chapters/第X-Y章-细纲.md（更新当前章节部分）
   ↓
7. 进入状态管理流程
```

#### 流程 5：规划类任务

```
用户："帮我写/改第N章的细纲" 或 "帮我写卷纲" 等
   ↓
根据任务类型调用对应 Agent：
├─ "写/改世界观" → Architect 或 WorldOrganizer
├─ "写/改卷纲" → VolumePlanner
├─ "写/改第X~Y章细纲" → OutlinePlanner（生成/修改一组细纲）
└─ "改规则" → BookRulesManager

每个 Agent：
1. 读取相关文件
2. 根据用户指令执行
3. 保存到对应文件
4. 主 Agent 验证输出
```

## 4. 规划类子 Agent

### 4.1 Architect（架构师）

| 字段 | 值                   |
| -- | ------------------- |
| 名称 | `novel-architect`   |
| 用途 | 新建小说时生成完整基础架构       |
| 模型 | opus                |
| 工具 | FileRead, FileWrite |

**职责**：

- 基于用户描述（书名、类型、概念）生成完整设定
- 输出：story-bible.md、volume-outline.md、book-rules.json、book-plan.md、current-state.md、pending-hooks.md

**融合点**：

- 来自其他项目 Agent 的 ArchitectAgent
- 融合老 Agent 中"长篇网文规划师"的设定撰写能力
- 增加"黄金三章"法则检查

### 4.2 VolumePlanner（卷纲规划师）

| 字段 | 值                             |
| -- | ----------------------------- |
| 名称 | `volume-planner`              |
| 用途 | 编写或修改全书卷纲                     |
| 模型 | sonnet                        |
| 工具 | FileRead, FileEdit, FileWrite |

**职责**：

- 读取 story-bible、current-state
- 根据用户指令编写/修改 volume-outline.md
- 保持与现有设定的一致性

**卷纲输出格式**：

```markdown
# 第一卷：[卷名]

## 卷概览
[本卷主要内容：主角遇到什么人，完成什么事情，对整体故事进度的推进]

## 章节分组规划

### 第 1~5 章：[阶段名称]
- 主线任务：[这5章的核心任务]
- 第1章：[剧情概要]
- 第2章：[剧情概要]
- 第3章：[剧情概要]
- 第4章：[剧情概要]
- 第5章：[剧情概要]

### 第 6~8 章：[阶段名称]
- 主线任务：[这3章的核心任务]
- 第6章：[剧情概要]
- 第7章：[剧情概要]
- 第8章：[剧情概要]

### 第 9~12 章：[阶段名称]
- 主线任务：[这4章的核心任务]
- 第9章：[剧情概要]
- ...

（以此类推，直到本卷结束）

---

# 第二卷：[卷名]
[同上]
```

### 4.3 OutlinePlanner（细纲规划师）

| 字段 | 值                             |
| -- | ----------------------------- |
| 名称 | `outline-planner`             |
| 用途 | 编写或修改章节细纲（每 5 章一组）            |
| 模型 | sonnet                        |
| 工具 | FileRead, FileEdit, FileWrite |

**职责**：

- 读取 story-bible、volume-outline、current-state、pending-hooks
- 支持三种模式：
  - `new`：为指定章节组（5章一组）新建细纲
  - `revise`：根据用户指令修改已有细纲组
  - `sync`：根据已写正文同步更新细纲（由 OutlineSyncer 调用）

**细纲输出格式**（每 5 章存放在同一个 MD 文件）：

```markdown
# 第 1~5 章 细纲

---

## 第1章：[章节名]

### 本章目标
[本章要达成的目标]

### 主要事件
1. [事件1：地点、起因、经过、结果]
2. [事件2]

### 出场人物
- [角色A]：（任务/作用）
- [角色B]：（任务/作用）

### 出场道具
- [道具A]：（作用）

### 伏笔处理
- 推进：[哪些伏笔要推进]
- 解决：[哪些伏笔要解决]

### 节奏安排
[开篇/中段/结尾的节奏]

### 场景设置
- 地点：
- 时间：
- 氛围：

---

## 第2章：[章节名]
[同上]

---

## 第3章：[章节名]
[同上]

---

## 第4章：[章节名]
[同上]

---

## 第5章：[章节名]
[同上]
```

**写正文时的工作方式**：

- 主 Agent 将整组（5章）细纲传递给 Writer
- Writer 每次只写一章，写完后走完整审核流程
- 然后再写下一章，直到该组全部写完

### 4.4 BookRulesManager（规则管理器）

| 字段 | 值                    |
| -- | -------------------- |
| 名称 | `book-rules-manager` |
| 用途 | 管理本书规则（硬规则、风格指南、约束）  |
| 模型 | haiku                |
| 工具 | FileRead, FileEdit   |

**职责**：

- 读取/修改 book-rules.json（JSON 格式）
- 支持新增、修改、删除规则

**book-rules.json 格式**：

```json
{
  "language": "zh",
  "hard_rules": [
    "不写H内容",
    "使用第一人称叙事",
    "不可违反的力量体系设定"
  ],
  "style_guidelines": [
    "口语化的对白推进剧情",
    "强台词、强情绪、强冲突",
    "长句短句相结合"
  ],
  "constraints": {
    "min_word_count": 1600,
    "max_word_count": 3000,
    "dialogue_ratio": 0.3
  },
  "banned_words": ["好像", "仿佛", "一丝", "未来", "光芒", "坚强", "一声", "一笑"]
}
```

## 5. 整理类 Agent（写作前准备）

### 5.1 WorldOrganizer（世界观整理）

| 字段 | 值                   |
| -- | ------------------- |
| 名称 | `world-organizer`   |
| 用途 | 整理当前章节需要的世界观设定      |
| 模型 | haiku               |
| 工具 | FileRead, FileWrite |

**职责**：

- 读取 story-bible.md
- 根据当前章节号，提取相关的世界观设定（当前场景相关的世界背景、力量体系等）
- 输出到 `02-正文/第N-M章/_task/世界观参考.md`

### 5.2 CharacterOrganizer（人物整理）

| 字段 | 值                     |
| -- | --------------------- |
| 名称 | `character-organizer` |
| 用途 | 整理当前章节出场的人物信息         |
| 模型 | haiku                 |
| 工具 | FileRead, FileWrite   |

**职责**：

- 读取 character-matrix.md、current-state.md
- 根据细纲中的出场人物，提取相关人物档案
- 输出到 `02-正文/第N-M章/_task/人物参考.md`

**融合点**：

- 来自老 Agent 的"小说人物总结助理"

### 5.3 ItemOrganizer（道具整理）

| 字段 | 值                   |
| -- | ------------------- |
| 名称 | `item-organizer`    |
| 用途 | 整理当前章节出场的道具和技能      |
| 模型 | haiku               |
| 工具 | FileRead, FileWrite |

**职责**：

- 读取 story-bible.md（道具/技能部分）
- 根据细纲中的出场道具，提取相关道具信息
- 输出到 `02-正文/第N-M章/_task/道具参考.md`

**融合点**：

- 来自老 Agent 的"道具查询助理"、"力量体系查询助理"、"小说道具和技能总结"

### 5.4 StorylineOrganizer（故事线整理）

| 字段 | 值                     |
| -- | --------------------- |
| 名称 | `storyline-organizer` |
| 用途 | 整理当前章节所在的故事线          |
| 模型 | haiku                 |
| 工具 | FileRead, FileWrite   |

**职责**：

- 读取 subplot-board.md、pending-hooks.md、volume-outline.md
- 根据当前章节号，匹配当前故事线覆盖范围
- 输出当前故事线内容到 `02-正文/第N-M章/_task/故事线参考.md`

**融合点**：

- 来自老 Agent 的"内容匹配助理"（匹配故事线）

### 5.5 ContextAssembler（上下文组装器）

| 字段 | 值                   |
| -- | ------------------- |
| 名称 | `context-assembler` |
| 用途 | 将各整理结果组装为写作任务汇总 MD  |
| 模型 | haiku               |
| 工具 | FileRead, FileWrite |

**职责**：

- 读取 `02-正文/第N-M章/_task/` 下的所有参考文件
- 读取当前章节所在的整组细纲（5章一组）
- 读取用户要求
- 读取 book-rules.json
- 组装为 `02-正文/第N-M章/_task/写作任务汇总.md`

**输出格式**：


**工作方式**：

- ContextAssembler 传递整组（5章）细纲给 Writer
- Writer 每次只写当前那一章（从任务文件中知道当前是第几章）
- 写完后走完整审核流程，再写下一章

**融合点**：

## 6. 写作流程 Agent

### 6.1 Writer（写手）

| 字段 | 值                   |
| -- | ------------------- |
| 名称 | `novel-writer`      |
| 用途 | 读取写作任务汇总，写正文        |
| 模型 | opus                |
| 工具 | FileRead, FileWrite |

**职责**：

- 读取 `02-正文/第N-M章/_task/写作任务汇总.md`
- 从任务文件中获取当前要写的章节号和整组细纲
- 从整组细纲中提取当前章节的细纲部分
- 根据其中的所有参考信息写正文
- 输出纯正文到 `02-正文/第N-M章/第N章.md`

**工作模式**：

- Writer 接收到的是整组（5章）细纲 + 当前章节号
- Writer 从整组细纲中定位到当前章节的细纲部分
- 写完后只输出当前章节的正文
- 下一轮再传入同样的整组细纲 + 下一个章节号

**写作要求**（融合老 Agent 的写作提示）：

```
1. 用口语化的对白推进剧情
2. 强台词、强情绪、强冲突、强对抗、强矛盾，增加看点
3. 长句短句相结合，避免通篇长句
4. 适当融入热梗与网络热词
5. 善用人物的小细节动作描述，增加画面感
6. 运用足够强烈的词组描写冲突
7. 单独句号注意换行，但不破坏句子结构
8. 开篇 30 字以内简洁句子
9. 结尾用简单动作或对话收尾，不展望未来
10. 避免使用禁用词列表
11. 对话占比 30% 左右
12. 字数 1600+ 字（可配置）
```

**输出规则**：

- 只输出纯正文内容
- 正文内不使用 Markdown 格式（加粗、列表、标题等）
- 不加章节标题
- 不使用状态更新标记
- 开篇直接进入剧情
- 结尾留合适的钩子
- 正文以 `.md` 文件保存，但文件内容是纯正文文本

**融合点**：

- 来自老 Agent 的"长篇网文写手"（详细写作风格要求）
- 来自其他项目 Agent 的 WriterAgent（文件操作规范）

**审计维度**（来自其他项目 Agent，28 项检查）：

### 8.3 CardManager（名片管理）

| 字段 | 值                  |
| -- | ------------------ |
| 名称 | `card-manager`     |
| 用途 | 更新小说名片（JSON）       |
| 模型 | haiku              |
| 工具 | FileRead, FileEdit |

**职责**：

- 读取 card.json
- 根据当前操作更新进度等信息
- 保存 card.json

**card.json 格式**：

```json
{
  "novel_id": "auto-generated",
  "title": "小说名称",
  "platform": "起点",
  "type": "玄幻",
  "concept": "一句话概括核心设定",
  "created_at": "2026-04-26",
  "current_chapter": 5,
  "target_chapter": 20,
  "total_chapters_planned": 500,
  "last_updated": "2026-04-26T18:30:00",
  "word_count": 45000,
  "status": "writing"
}
```

**关键字段说明**：

- `current_chapter`：当前已写完的章节号（纯数字），`5` 表示第1-5章已完成
- `target_chapter`：用户要求写到的目标章节号（纯数字），自动续传功能以此为基准判断 `target_chapter > current_chapter` 时触发
- `total_chapters_planned`：计划总章节数
- `word_count`：当前总字数（中文汉字）
- `status`：`planning` / `writing` / `completed` / `paused`
