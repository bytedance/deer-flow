# 一期后端需求文档：小说标签系统

> 目标：为对话系统提供小说标签的 API 支持，包括标签枚举获取和标签数据存储

---

## 1. 需求概述

### 1.1 功能目标
- 提供 API 接口，扫描并返回所有可用的小说标签（从 workspace/book 目录）
- 利用现有的全局变量系统存储标签
- 支持在查询对话列表时返回小说标签信息

### 1.2 技术约束
- 复用现有的全局变量系统（已开发完成）
- 标签存储为对话级全局变量，`is_system=True`，`llm_editable=False`
- 保持与现有全局变量系统的完全兼容
- **不需要标签验证**：允许任意标签值
- **数据量不大**：当前不需要复杂的性能优化，简单缓存即可

---

## 2. 技术方案

### 2.1 标签数据来源

标签来源于所有对话工作目录下的 `workspace/book` 文件夹：

```
{base_dir}/
└── threads/
    └── {thread_id}/
        └── user-data/
            └── workspace/
                └── book/
                    ├── 小说A/
                    ├── 小说B/
                    └── 小说C/
```

标签枚举值 = `book` 目录下所有一级文件夹的名称集合

**重要说明**：`book` 目录下的文件夹由 Agent 在运行时自动创建，前端只负责读取现有目录作为标签枚举值。

### 2.2 数据存储方案

**利用现有的全局变量系统**（已开发完成）：

存储位置：`{DEER_FLOW_HOME}/threads/{thread_id}/variables.json`

调用方式（前端 onCreated 回调中）：
```
PUT /api/global-variables/threads/{thread_id}/novel_tag
{
  "value": "小说A",
  "description": "当前正在进行的小说",
  "is_system": true,
  "llm_editable": false
}
```

**注意**：
- `novel_tag` 是对话级全局变量
- 设置为系统变量（`is_system=True`），前端只读，不可编辑
- 大模型不可修改（`llm_editable=False`）
- 不选标签时不创建变量

## 3. API 设计

### 3.1 获取小说标签列表

**接口路径**：`GET /api/novel-tags`

**请求参数**：无

**响应格式**：
```json
{
  "tags": ["小说A", "小说B", "小说C"]
}
```

**实现逻辑**：
1. 扫描 `{base_dir}/threads/` 下所有线程目录
2. 检查每个线程的 `user-data/workspace/book/` 目录
3. 收集 `book` 目录下所有一级文件夹名称
4. 去重、排序后返回
5. **使用 5 分钟简单缓存**，避免频繁扫描

**代码实现**：

```python
# backend/app/api/routes/novel_tags.py

from fastapi import APIRouter, HTTPException
from deerflow.config.paths import get_paths
import logging
import time

logger = logging.getLogger(__name__)
router = APIRouter()

# 简单缓存
_novel_tags_cache = {
    "tags": [],
    "last_updated": 0,
    "ttl": 300  # 5 分钟缓存
}

@router.get("/api/novel-tags")
async def get_novel_tags() -> dict:
    """
    获取所有可用的小说标签
    
    扫描所有线程的工作目录下的 book 文件夹，
    返回所有小说文件夹名称作为标签枚举值
    """
    current_time = time.time()
    
    # 检查缓存是否有效
    if (
        _novel_tags_cache["tags"]
        and current_time - _novel_tags_cache["last_updated"] < _novel_tags_cache["ttl"]
    ):
        return {"tags": _novel_tags_cache["tags"]}
    
    try:
        paths = get_paths()
        base_dir = paths.base_dir
        threads_dir = base_dir / "threads"
        
        if not threads_dir.exists():
            return {"tags": []}
        
        tags = set()
        
        # 遍历所有线程目录
        for thread_dir in threads_dir.iterdir():
            if not thread_dir.is_dir():
                continue
            
            # 检查 book 目录
            book_dir = (
                thread_dir / "user-data" / "workspace" / "book"
            )
            
            if book_dir.exists() and book_dir.is_dir():
                # 收集所有一级文件夹名称
                for item in book_dir.iterdir():
                    if item.is_dir():
                        tags.add(item.name)
        
        sorted_tags = sorted(list(tags))
        
        # 更新缓存
        _novel_tags_cache["tags"] = sorted_tags
        _novel_tags_cache["last_updated"] = current_time
        
        return {"tags": sorted_tags}
        
    except Exception as e:
        logger.error(f"Failed to get novel tags: {e}")
        raise HTTPException(
            status_code=500,
            detail=f"Failed to get novel tags: {str(e)}"
        )
```

## 4. 缓存设计

### 4.1 缓存策略

采用**简单 TTL 缓存**：

```python
_novel_tags_cache = {
    "tags": [],
    "last_updated": 0,
    "ttl": 300  # 5 分钟缓存
}
```

**优点**：
- 实现简单
- 适合当前数据量不大的场景
- 减少文件系统扫描频率

**缺点**：
- Agent 创建新小说目录后，最多需要等待 5 分钟才能在标签选择器中看到
- 无法实时反映文件系统变化

**后续优化方向**（如果需要）：
- 添加手动刷新缓存的 API
- 使用文件系统监听（如 `watchdog`）自动刷新

### 4.2 缓存失效场景

以下情况会导致缓存过期：
1. TTL 超时（5 分钟）
2. 服务重启（缓存在内存中）

## 5. 错误处理

### 5.1 异常场景

| 场景 | 处理方式 |
|------|---------|
| book 目录不存在 | 返回空标签列表 `{"tags": []}` |
| 权限问题 | 记录错误日志，返回空列表 |
| 目录结构异常 | 跳过异常目录，继续扫描 |
| 扫描超时 | 返回缓存数据（如果有） |

### 5.2 日志记录

```python
import logging

logger = logging.getLogger(__name__)

# 记录扫描开始
logger.info(f"Scanning novel tags from {threads_dir}")

# 记录扫描结果

# 记录缓存命中

# 记录错误
```
