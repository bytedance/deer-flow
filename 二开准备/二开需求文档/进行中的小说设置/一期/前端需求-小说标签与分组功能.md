# 一期需求文档：进行中的小说设置 - 标签系统与分组功能

> 目标：为对话系统添加小说标签功能，支持在新建对话时选择小说标签，并在对话列表中按标签分组展示

---

## 1. 需求概述

### 1.1 功能目标
- 在新建对话时，允许用户从枚举列表中选择当前正在进行的小说标签（非必填）
- 标签枚举值来源于工作目录下 `workspace/book` 文件夹中的所有文件夹名称
- 标签选择后存储为**对话级别的全局变量**（系统变量），后续可读取
- 在左侧对话列表页面增加按标签分组的功能

### 1.2 业务规则

1. **标签只能在创建新对话时选择**，创建后不可修改
2. **标签为非必填项**：用户可以不选标签，此时不创建变量
3. **标签值来源**：扫描 `{base_dir}/threads/{thread_id}/user-data/workspace/book/` 下的所有一级文件夹名称
4. **标签存储**：存储为对话级别的全局变量（系统变量），`is_system=True`，`llm_editable=False`
5. **分组展示**：
   - 左侧对话列表按标签分组显示
   - 未设置标签的对话归为"未分类"分组
   - **每次打开页面时，所有分组默认折叠状态，不保留用户的展开/折叠偏好**
6. **目录来源说明**：`book` 目录下的文件夹由 Agent 在运行时自动创建，前端只需要读取现有的目录作为标签枚举值

```typescript
const groupedThreads = useMemo(() => {
  const groups: Record<string, AgentThread[]> = {};
  threads.forEach(thread => {
    // 从对话级全局变量中读取 novel_tag
    const tag = thread.globalVariables?.novel_tag || "未分类";
    if (!groups[tag]) {
      groups[tag] = [];
    }
    groups[tag].push(thread);
  });
  return groups;
}, [threads]);

// 分组展开/折叠状态（使用 Map 管理多个分组的状态）
const [expandedGroups, setExpandedGroups] = useState<Set<string>>(new Set());

// 切换分组展开/折叠
const toggleGroup = useCallback((groupName: string) => {
  setExpandedGroups(prev => {
    const next = new Set(prev);
    if (next.has(groupName)) {
      next.delete(groupName);
    } else {
      next.add(groupName);
    }
    return next;
  });
}, []);
```

### 4.3 API 调用

#### 4.3.1 获取标签列表

```typescript
// frontend/src/core/novel-tags/hooks.ts

import { useQuery } from "@tanstack/react-query";
import { getBackendBaseURL } from "@/core/config";

export function useNovelTags() {
  return useQuery<{ tags: string[] }>({
    queryKey: ["novel-tags"],
    queryFn: async () => {
      const response = await fetch(`${getBackendBaseURL()}/api/novel-tags`);
      if (!response.ok) {
        throw new Error("Failed to fetch novel tags");
      }
      return response.json();
    },
    staleTime: 5 * 60 * 1000,  // 5 分钟内不重新请求
  });
}
```

#### 4.3.3 加载对话列表时读取标签

需要获取每个对话的全局变量：

```typescript
// 在加载对话列表时，并行获取每个对话的变量
const loadThreadsWithNovelTags = async (threads: AgentThread[]) => {
  const threadsWithTags = await Promise.all(
    threads.map(async (thread) => {
      const response = await fetch(
        `${getBackendBaseURL()}/api/global-variables/threads/${thread.thread_id}`
      );
      if (response.ok) {
        const data = await response.json();
        const novelTagVar = data.variables?.find((v: any) => v.key === "novel_tag");
        return {
          ...thread,
          globalVariables: {
            novel_tag: novelTagVar?.value,
          },
        };
      }
      return { ...thread, globalVariables: { novel_tag: undefined } };
    })
  );
  return threadsWithTags;
};
```

## 5. 后端技术方案

### 5.1 新增 API 端点

#### 5.1.1 获取小说标签列表

文件：`backend/app/api/routes/novel_tags.py`

```python
from fastapi import APIRouter, HTTPException
from deerflow.config.paths import get_paths
import logging
import time

logger = logging.getLogger(__name__)
router = APIRouter()

# 简单缓存
_novel_tags_cache = {
    "tags": [],
    "last_updated": 0,
    "ttl": 300  # 5 分钟缓存
}

@router.get("/api/novel-tags")
async def get_novel_tags():
    """
    获取所有可用的小说标签
    
    扫描所有线程的工作目录下的 book 文件夹，
    返回所有小说文件夹名称作为标签枚举值
    """
    current_time = time.time()
    
    # 检查缓存是否有效
    if (
        _novel_tags_cache["tags"]
        and current_time - _novel_tags_cache["last_updated"] < _novel_tags_cache["ttl"]
    ):
        return {"tags": _novel_tags_cache["tags"]}
    
    try:
        paths = get_paths()
        base_dir = paths.base_dir
        threads_dir = base_dir / "threads"
        
        if not threads_dir.exists():
            return {"tags": []}
        
        tags = set()
        
        # 遍历所有线程目录
        for thread_dir in threads_dir.iterdir():
            if not thread_dir.is_dir():
                continue
            
            # 检查 book 目录
            book_dir = (
                thread_dir / "user-data" / "workspace" / "book"
            )
            
            if book_dir.exists() and book_dir.is_dir():
                # 收集所有一级文件夹名称
                for item in book_dir.iterdir():
                    if item.is_dir():
                        tags.add(item.name)
        
        sorted_tags = sorted(list(tags))
        
        # 更新缓存
        _novel_tags_cache["tags"] = sorted_tags
        _novel_tags_cache["last_updated"] = current_time
        
        return {"tags": sorted_tags}
        
    except Exception as e:
        logger.error(f"Failed to get novel tags: {e}")
        raise HTTPException(
            status_code=500,
            detail=f"Failed to get novel tags: {str(e)}"
        )
```

## 6. 业务流程

### 6.1 创建对话流程

```
用户点击"新建对话"
    ↓
前端获取小说标签列表（GET /api/novel-tags）
    ↓
显示标签选择器（非必填，可选：清空/小说A/小说B/...）
    ↓
用户选择标签（或不选）
    ↓
用户输入消息并提交
    ↓
前端创建线程
    ↓
在 onCreated 回调中：
  - 如果选择了标签，调用 PUT /api/global-variables/threads/{thread_id}/novel_tag
  - 设置为系统变量（is_system=True, llm_editable=False）
    ↓
对话创建完成，左侧列表刷新显示
```

### 6.2 对话列表分组流程

```
前端加载对话列表
    ↓
对每个对话，获取其全局变量（GET /api/global-variables/threads/{thread_id}）
    ↓
提取 novel_tag 变量值
    ↓
按 novel_tag 分组（undefined 归为"未分类"）
    ↓
渲染分组列表，所有分组默认折叠
    ↓
用户可点击分组标题展开/折叠
    ↓
刷新页面时，重新折叠所有分组
```

## 9. 注意事项

1. **缓存策略**：标签列表使用 5 分钟简单缓存，缓存过期后自动刷新
2. **向后兼容**：已有对话没有 novel_tag 变量，前端需做兼容处理（归为"未分类"）
3. **国际化**：标签选择器和分组标题需要支持 i18n
4. **用户体验**：标签选择器应明确标注"可选"，避免用户困惑
5. **分组排序**：建议按字母顺序或创建时间排序分组，"未分类"固定放在最后
6. **系统变量特性**：novel_tag 设置为 `is_system=True`，`llm_editable=False`，前端只读展示，大模型不可修改
