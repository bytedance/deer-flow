# 需求总览：进行中的小说设置

> 目标：为对话系统添加小说标签功能，实现对话按小说分组管理，并为后续的全局变量注入和目录限制提供基础

---

## 1. 需求背景

在多小说创作场景中，需要：
- 为每个对话标记所属的小说项目
- 在对话列表中按小说分组展示
- 让 Agent 知道当前正在编辑哪部小说
- 限制文件浏览器只能操作当前小说目录

### 1.1 前置依赖：共享文件夹路径全局变量

**背景**：
项目 `config.yaml` 中配置了一个共享文件夹：
```yaml
sandbox:
  mounts:
    - host_path: c:\xiangmu\deer-flow\backend\.deer-flow\shared-data
      container_path: /mnt/shared-data
      read_only: false
```

**需求**：
1. 系统启动时读取 `config.yaml` 的 `sandbox.mounts` 配置
2. 自动初始化项目级全局变量 `shared_data_path="/mnt/shared-data"`
3. 将该变量注入到所有 Agent 的提示词中
4. 配置修改后重启服务，变量值自动更新

**用途**：
- 所有 Agent 都需要知道共享文件夹路径才能正确读写文件
- 作为全局变量系统的首批变量，验证注入机制

## 4. 关键技术点

### 4.1 标签来源
- 扫描所有线程的 `workspace/book` 目录
- 收集所有一级文件夹名称
- 去重、排序后作为枚举值

### 4.2 数据存储
- 使用对话级全局变量系统（已开发完成）
- 变量名：`novel_tag`
- 类型：对话级系统变量（`is_system=True`, `llm_editable=False`）
- 存储位置：`{DEER_FLOW_HOME}/threads/{thread_id}/variables.json`
- 可选值：string 或 undefined（未设置时不创建变量）

### 4.3 前端展示
- 新建对话时显示标签选择器
- 对话列表按标签分组
- 分组支持展开/折叠（不保留状态）

### 4.4 二期注入（概要）
- 提示词模板变量：`{novel_tag}`
- 资源管理器根目录：`/mnt/user-data/workspace/book/{novel_tag}/`

## 7. 风险与注意事项

### 7.1 性能
- 一期标签扫描可能较慢，需要实现缓存机制
- 缓存 TTL 为 5 分钟
- 当前数据量不大，无需复杂优化

### 7.2 兼容性
- 已有对话没有 novel_tag，前端需兼容显示为"未分类"
- 后端 API 需处理 novel_tag 为空的情况

### 7.3 安全性
- 二期目录限制必须在后端强制验证
- 不能仅依赖前端 UI 限制

### 7.4 业务规则
- **标签非必填**：用户可以不选标签
- **标签不可修改**：创建后锁定，只有新建对话时可以选择
- **不保留分组状态**：每次打开页面，所有分组默认折叠
- **后端不验证标签**：允许任意标签值
- **标签为系统变量**：`is_system=True`，前端只读展示
- **大模型不可修改**：`llm_editable=False`
